# 랜딩 페이지 프로젝트 — 고정 컨텍스트

## 이 사이트는 무엇인가
취준생 시청자용 포트폴리오 랜딩 페이지. 채널(스레드·틱톡·인스타) 발견 → 착지 → 채널로 흘려보내는 동선. C레벨용 아님(그건 별도 PDF).

## 절대 원칙 (어기면 결이 무너짐)
- 회사명·브랜드명·구체 매출수치·내부 정보 노출 0. (데이터 정직성·마스킹)
- 톤: 히어로는 능청·블랙코미디(자기객관화), 아래로 갈수록 실무 톤. 비장·감성 금지.
- "빈 상태"를 숨기지 않는다. 칩 자산은 0개에서 시작. 비어 있음을 콘셉트로 드러낸다("매주 채워가는 중"). 가짜로 채우지 말 것.
- 완벽주의로 발행을 늦추지 않는다. MVP 최소 구동 → 점진 업그레이드.

## 기술 스택
- 정적 HTML / CSS / 바닐라 JS. 프레임워크(React 등) 쓰지 않는다.
- 이미지는 별도 파일 + 경로 참조. Base64 인라인 임베드 금지.
- 영상은 플랫폼(유튜브 등) iframe 임베드. 영상 파일 직접 호스팅 금지.
- 배포: Cloudflare Pages (Git 기반). HTML/CSS/JS 파일은 항상 분리 유지.

## 페이지 구조 (위→아래)
① 히어로 → ② 영상 임베드 → ③ 대문 길안내(띠) → ④ 칩 A/B(가로) → ⑤ 위젯 자리 → ⑥ 채널 링크
설계 동선: 착지 → 영상 후크 → 길안내 → 칩 자기주도 탐색 → 채널로 이탈.

## 확정 카피는 함부로 바꾸지 않는다
헤드라인·길안내·칩 라벨은 Module 03 확정본. 수정이 필요하면 사용자에게 먼저 확인.
